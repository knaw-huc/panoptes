# Panoptes

Panoptes is a multi-tenant dataset browser and search tool. This repository contains the back-end code.

## Introduction

The Panoptes back-end is an API for searching through Elasticsearch indices and retrieving detailed information about specific
records using configurable data sources (so far the only supported option is the Clariah CMDI editor API, but more will be added in the future).

The API is made using FastAPI, dataset configuration is done in a Mongo database.

## Quick start

### Installation

This software is not released yet. Container images will be available here on GitHub. In order to use the latest version, you can build the container
image yourself. There is a Docker Bake configuration file. In order to set the name of the image, crate your own `docker-bake.override.hcl` file to set two variables:

```hcl
IMAGE = "some-registry.com/your-image-name"
TAG = "beta-1"
```
And then build
```shell
docker buildx bake
```

### Usage

As this is a multi-tenant service, you should configure the domain names belonging to the tenants in the database configuration. Then, it's possible to add
multiple datasets per tenant. 

## MongoDB configuration
The configuration for the tenants and datasets in Panoptes is managed in MongoDB. In order to configure this, you need to have at least two databases:

- a database named `main`.
- a database named after your tenant.

## main
The `main` database uses only one table / collection. This collection is called `tenants` and contains the configuration of the tenants using this system.

### Tenants
It should contain the following data:

| Column | Explanation                                                                                                                      | Example                  |
| ------ | -------------------------------------------------------------------------------------------------------------------------------- | ------------------------ |
| _id    | Mongo ID. This is automatically set                                                                                              | 673e01aff777cc01f84c4244 |
| name   | Name of the tenant. This should be the same as the name of the database created for this tenant.                                 | `tenant-a`               |
| domain | The domain name which corresponds to this tenant. API calls received on this domain will be treated as belonging to this tenant. | `tenant-a.example.com`   |

## {tenant-name}
This database contains the configuration for a specific tenant. You need as many of those as you have tenants in your system. It contains more collections: `datasets`, `detail_properties`, `facets` and `result_properties`.

### Collection `datasets`
This contains the configuration for the different datasets are available to a tenant.

| Column                         | Explanation                                                                                                                                                                                           | Example                                                                                                      |
| ------------------------------ |-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------| ------------------------------------------------------------------------------------------------------------ |
| _id                            | Mongo ID (autogenerated).                                                                                                                                                                             | 673e01aff777cc01f84c4244                                                                                     |
| tenant_name                    | The name of the tenant. Should match the one in the `tenants` collection.                                                                                                                             | `tenant-a`                                                                                                   |
| name                           | The name of the dataset. Should be URL safe.                                                                                                                                                          | `my-dataset`                                                                                                 |
| es_index                       | The name of the ElasticSearch index that contains this dataset.                                                                                                                                       | `my-dataset-data`                                                                                            |
| data_type                      | `elasticsearch` or `cmdi`. Defines what type of data is used for the detail views of this dataset. Options are just elasticsearch which uses data from the index also used for search, and            | `elasticsearch`                                                                                                    |
| data_configuration             | This is an object containing configuration specific to the `data_type`                                                                                                                                | `{}`                                                                                                         |
| data_configuration.id_property | For data_type `cmdi`: the property in the ES index which contains the ID used in the data store used (cmdi)<br/><br/>For data_type `elasticsearch`: empty                                             | `id` or empty for `elastic`                                                                                  |
| data_configuration.base_url    | For data_type `cmdi`: the base URL of the editor API where the data can be found, up to `/record` so the `id_property` value can be inserted after that<br/><br/>For data_type `elasticsearch`: empty | `https://editor-domain-name.example.com/app/my-dataset/profile/clarin:p_12345/record` or empty for `elastic` |
| detail_id                      | The field in the results / es_index by which to retrieve one specific item.                                                                                                                           | `id`                                                                                                         |

### Collection `detail_properties`
Contains information for what properties to render in the details view.

| Column       | Explanation                                                                                                                                                                                                                                                                                       | Example                |
| ------------ |---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------|
| _id          | MongoDB id (autogenerated).                                                                                                                                                                                                                                                                       |                        |
| dataset_name | The name of the dataset this belongs to.                                                                                                                                                                                                                                                          | `my-dataset`           |
| name         | Name of the property.                                                                                                                                                                                                                                                                             | `es_data`              |
| type         | <ul><li>`cmdi`</li><li>`list`</li></ul>Both render a JSON object with key/value pairs, but `cmdi` is specific for items using the CMDI editor as back-end.<br/><br/><ul><li>`json`</li></ul>Renders a JSON object according to typing information present in the `schema` object of the response. | `cmdi`', `list`, `json` |
| path         | JSONPath expression for where to find the property in the ElasticSearch find details.                                                                                                                                                                                                             | `$`                    |
| order        | Used for sorting the properties on the details page.                                                                                                                                                                                                                                              | `0`                    |

### Collection `facets`
The facets to use on the search page

| Column       | Explanation                                                                                                                                                               | Example          |
| ------------ |---------------------------------------------------------------------------------------------------------------------------------------------------------------------------| ---------------- |
| _id          | MongoDB id (autogenerated).                                                                                                                                               |                  |
| dataset_name | The name of the dataset this belongs to.                                                                                                                                  | `my-dataset`     |
| name         | Readable name of the facet.                                                                                                                                               | Folder           |
| property     | ES property this facet filters on.                                                                                                                                        | `folder.keyword` |
| type         | `text`, `tree` or `range` for now. The type of facet this is. Text is a list of checkboxes, tree the same but hierarchical and range is a slider with a min and max value | `tree`           |

### Collection `result_properties`
The properties to use for the search results. This requires at least an ID, or field by which to retrieve the details of an item with (see "datasets"). Other fields rendered in the front-end application are `title`, `description` and `tags` (the latter being optional).

| Column       | Explanation                                                      | Example |
| ------------ | ---------------------------------------------------------------- | ------ |
| _id          | MongoDB id                                                       |        |
| dataset_name | The name of the dataset this belongs to                          | `my-dataset` |
| name         | The name of this field                                           | `title` |
| path         | JSONPath expression of where to find this field in the ES result | `$`  |
| order        | The order in which to render the properties                      | `0`    |

### Collection `schema`
Schema data that is collected when generating a details response. 

| Column       | Explanation                                                                                        | Example                                                                                                                                                                                 |
|--------------|----------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| _id          | MongoDB id (autogenerated).                                                                        |                                                                                                                                                                                         |
| dataset_name | The name of the dataset this belongs to.                                                           | `my-dataset`                                                                                                                                                                            |
| order        | The order in which to render the schema properties (obsolete).                                     |                                                                                                                                                                                         |
| property     | The name of the property for which schema data is encoded.                                         | `title`, `uitgever`                                                                                                                                                                     |
| schema      | The schema object describing the property contents. This is as much JSONSchema-like as possible.   | `{"type": "string", "format": "link", "x-link-to": "/politieke-tijdschriften-uitgever_drukker/details/$uitgeverId"}`<br/><br/>`{"type": "string", "format": "markdown-html"}`<br/><br/>`{"x-omit": true}` |

Recognized schema properties for a scalar value:

- `format`: one of `{link, markdown-html}`, where:
  - `link` renders a link and should be accompanied by a `x-link-to` property. 
  - `markdown-html` renders textual content in a (React) Markdown component (nicer for larger text blobs).
- `x-omit`: should the JSON renderer actually render the value?

### Collection `translations`
This collections contains the translations for datasets and locale(s).

| Column | Explanation                                                       | Example             |
|-------|--------------------------------------------------------------------|---------------------|
| _id   | MongoDB id  (autogenerated).                                       |                     |
| label_key | The name of the dataset this belongs to                        | `label_key`         |
| label_value | The name of this field                                       | `Mijn label waarde` |
| locale | The locale for which the (label_key, label_value) pair is defined.| `NL`, `EN`          |

The structure above enables re-use of labels across multiple tenants and data sets.
